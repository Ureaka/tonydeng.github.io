发表于 2015-01-05   |   分类于 程序设计   |  

松耦合

当你能够做到修改一个组件而不需要更改其他组件时，就做到了松耦合，松耦合对于代码可维护性来说是至关重要的。

原则

  • 不要使用 css 表达式。（这种方式应该已经废弃了）
  • javascript 和 css 之间只通过 className 进行通信。
  • 不要使用 html 的 on 属性，如：onclick。应该使用 id 保持 javascript 和 html 的沟通。
  • 使用模板。
